# Product

## Register

product

## Users

Groups of 4–10 friends playing a social deduction party game together, typically on separate phones or laptops in the same room. Players are engaged, competitive, and want the game to feel like they've stepped into another world — not just a generic game UI.

## Product Purpose

Shelter is a real-time multiplayer adaptation of the board game "Bunker/Бункер." A catastrophic scenario is selected each session, and the game should make players feel immersed in that specific world. Each scenario is its own alternate reality: a Soviet nuclear aftermath, a bio-containment outbreak, a NASA mission going wrong.

## Brand Personality

Cinematic, world-specific, high-stakes. The product should feel like entering a different reality for each catastrophe.

## Anti-references

- Generic dark-purple glowing game UIs (the previous design)
- SaaS product aesthetics applied to a social game
- Any design that looks and feels the same regardless of which catastrophe was selected

## Design Principles

1. Each scenario has its own world — the UI is inside that catastrophe's reality, not floating above it
2. Legibility under pressure — game state must be crystal clear; atmosphere is a layer on top of readability
3. Texture over generic glow — physical, tactile design materials beat digital neon
4. Typography as world-building — font weight, scale, and rhythm signal which reality you're in
5. Radius as personality — corners carry meaning: 0px is totalitarian and brutal; 6px is engineered and precise

## Accessibility & Inclusion

WCAG AA minimum. All scenario themes must maintain ≥4.5:1 contrast for body text, ≥3:1 for large display text. Motion can be atmospheric but must not obscure game state. Reduced motion alternative required for all animations.
